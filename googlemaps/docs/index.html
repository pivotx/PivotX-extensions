Google Maps Extension Docs

This extension enables you to display a map from Google Maps in your PivotX blog. Additionally, you can add an overlay on the map defined by a KML-file.

The extension uses the Google Maps (JavaScript) API. This has many benefits compared to using an iframe as the normal map embedding does. The biggest benefit being the ease of adding (KML) overlays.

Usage

Setup and configuration

The extension consists of an admin page and a snippet. Before starting to use the snippet (but after enabling), you must visit the "Google Maps" tab on the "Configuration" and set the correct Google Maps API key for your domain. On the same form you can also set the default size of the map, map zoom level and latitude and longiture (x and y coordinates).

Snippet syntax

The minimal usage is:

[[ googlemaps ]]

The snippet takes six optional parameters:

  • overlay - an absolute URL to an offsite KML file or the name of a KML file you have uploaded to the media folder.
  • width - the width of the map, overriding the default value.
  • height - the height of the map, overriding the default value.
  • zoom - the zoom level of the map, overriding the default value.
  • lat - the latitude of the (centre of the) map, overriding the default value.
  • long - the longitude of the (centre of the) map, overriding the default value.

A complex usage example replicating an example from the Google Maps documentation:

[[ googlemaps overlay=http://mapgadgets.googlepages.com/cta.kml height=480 width=640 lat=41.875696 long=-87.624207 zoom=11 ]]

Tips

  • To upload KML files (using the Media Manager) add application/vnd.google-earth.kml+xml to the list of allowed file types in the PivotX configuration.
