Google Maps (Version 3) Extension Docs

This extension enables you to display a map from Google Maps in your PivotX blog. Additionally, you can add an overlay on the map defined by a KML-file.

The extension uses the Google Maps (JavaScript) API version 3. This has many benefits compared to using an iframe as the normal map embedding does. The biggest benefit being the ease of adding (KML) overlays.

Usage

Setup and configuration

The extension consists of an admin page and a snippet. Before starting to use the snippet (but after enabling), you should visit the "Google Maps (Version 3)" tab on the "Configuration" and set the default size of the map, map zoom level and latitude and longitude (x and y coordinates). You don't need an API key in version 3 of the Google Maps API.

Snippet syntax

The minimal usage is:

[[ googlemaps_v3 ]]

The snippet takes six optional parameters:

  • maptype - the type of map to use, overriding the default type. Possible values are "roadmap", "satellite", "hybrid" and "terrain".
  • overlay - an absolute URL to an offsite KML file or the name of a KML file you have uploaded to the media folder.
  • kml_preserveviewport - whether the viewport should be preserved or not when adding an overlay (defined by the KML file).
  • width - the width of the map, overriding the default value.
  • height - the height of the map, overriding the default value.
  • zoom - the zoom level of the map, overriding the default value.
  • lat - the latitude of the (centre of the) map, overriding the default value.
  • long - the longitude of the (centre of the) map, overriding the default value.

Tips

  • If you add an overlay, you don't have to set the zoom, latitude and longitude since the Javascript code will try to determine the correct location/size.
  • To upload KML files (using the Media Manager) add application/vnd.google-earth.kml+xml to the list of allowed file types in the PivotX configuration.
  • This extension uses jQuery for the onload handling. If you have set never_jquery in the Advanced Configuration, the maps won't load.

Limitations

  • The KML-file must be located on the server - displaying external KML files doesn't work.
  • Only markers, not polygons, are supported in the KML files.
